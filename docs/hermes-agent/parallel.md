# §08 并行与子代理：delegate_task

Hermes 可以把一部分工作“外包”给子代理并行执行。  
这节之后，你才真正拥有“团队助手”，而不是“一对一聊天机器人”。

---

## Case

同一份资料源，你希望同时产出：
1. 一份摘要（简体、300 字）
2. 一份评论（挑刺、指出潜在误导）

两个任务互相独立，应该同时跑，而不是串行。

---

## Follow

```text
你继续回复 /tmp/input.md 里的文章，
我同时启动两个子代理：
1. 内容摘要，300字简体
2. 内容评判，列出潜Inaccuracy
用 delegate_task 并行执行并把结果分别写入 /tmp/summary.md and /tmp/review.md
```

---

## 并行时的常见结果

| 模式 | 说明 |
|------|------|
| summary 先完成 | 正常，不等 review |
| review 先完成 | 正常，不等 summary |
| 两者内容冲突 | 分别渲染，最后你人工判断 |
| 单个子代理失败 | 另一个子代理任务不受影响 |

---

## 生成者 vs 评判者

- 生成者越专注任务，评判者越偏向审查
- 同一个代理既当运动员又当裁判，结果会变松
- 让两个角色分开，不是逼它们更极端，而是让复核真正有效

---

## 注意点

- `delegate_task` 是**同步阻塞**的：子代理没跑完，主任务继续等待结果
- 超出 tool 上限会提前终止
- 不要提交会让子代理再触发新的子代理的任务，保持层级清楚
