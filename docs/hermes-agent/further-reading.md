# §10 进阶阅读

这一节不是“改变世界的终极指南”，而是给已经上手的你继续深挖的路线图。

---

## 推荐方向

- 按自己的使用习惯做 **Skill**，而不是依赖他人 Skill；维护会更顺利
- 关注官方的 **Loop Engineering** 思路：不要把 Hermes 当作一对一工具，而要把它的 Docker 版、Gateway、Cron 纳入同一套自动循环
- 如果你同时在用 Claude Code / Codex / OpenClaw，建议各选一个**交叉验证的测试用例**，去找各自的边界

## 写在最后

好的工具不会取代你想清楚目标。Hermes 负责的是**把重复的活干净执行**，而决定要做什么、为什么做这件事，仍然在你手上。
